## Findings + Disposition

### MED-1 — 025b/c-Präzedenz für die docker-run-Analyse falsch zitiert
**Fund:** Die 025b/c-Reader-Belege (poppler/PIL) liefen bewusst **außerhalb** des gepinnten Images als Reviewer-Cross-Check — das Gegenteil des behaupteten Musters. Die Regelkonformität des Dry-Runs trägt eine andere Begründung.
**Disposition: eingearbeitet** — Begründung auf die tragenden Gründe umgestellt (clang-tidy **im Image gepinnt** → §2.9-Intention erfüllt; Guard erlaubt `docker run img <tool>`; kein behaupteter Gate → gate-consistency unberührt); Fehl-Präzedenz explizit als solche benannt.

### MED-2 — Sub-Area-Modus deckte die Fix-Pfade (src/**, plugins/**) nicht
**Disposition: eingearbeitet** — §7 um die Sub-Areas der Lint-Fixes ergänzt (Hexagon-Kern/Adapter/Plugin-Host, je GF; Fixes als verhaltensneutrale Randänderung unter der „kein Verhaltens-Umbau"-Grenze; Plugin-Host als benannter Kollisions-Fokus).

### MED-3 — „Checks/WarningsAsErrors synchron" beschrieb nicht den realen Fatalitäts-Mechanismus
**Fund:** Die lint-Stage promotet per CLI `--warnings-as-errors='*'` jeden aktivierten Check **sofort** zum Fehler — der File-Block ist redundant; eine „Beobachtungsphase" via Checks-only-Aktivierung gibt es nicht.
**Disposition: eingearbeitet** — DoD auf die reale Mechanik gestellt (Aktivierung nur **nach** Fix; WarningsAsErrors-Sync als Hygiene, nicht Mechanismus).

### LOW-1 — HeaderFilterRegex `^src/` unbenannt (plugins/-Header außerhalb Gate & Matrix)
**Disposition: eingearbeitet** (§6-Vorbehalt: Matrix ist gate-repräsentativ, da der Dry-Run dieselbe .clang-tidy erbt; Lücke benannt).

### LOW-2 — Exakte docker-run-Kommandos nicht als Beleg fixiert
**Disposition: eingearbeitet** (DoD: Kommandos werden im Beleg-Report mitprotokolliert).

### LOW-3 — Umfangs-Deckel pro Check, nicht kumulativ
**Disposition: eingearbeitet** (kumulativer Fix-Budget-Escape ~60: priorisieren, Rest als Folge-Kandidat/Split).

### LOW-4 — Dryrun-Beleg unter docs/reviews/ ist eine neue Artefakt-Klasse
**Disposition: eingearbeitet als benannte Ausnahme** (Evidenz-Beleg statt Review; Ort wegen ids-Exempt-Klasse).
