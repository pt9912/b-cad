## Findings

### MED-1 (frisch) — Re-Exklusions-Klausel technisch unpräzise
`.clang-tidy` `Checks:`/`WarningsAsErrors:` schalten Check-**Namen global** (alle
Dateien); eine „einzelne per-Fundstelle-Re-Exklusion in `.clang-tidy`" existiert
nicht. Realistischer Fallback bei einem false-positive: (i) Alternativ-Fix
(bevorzugt), (ii) dokumentiertes Wieder-Deaktivieren des **ganzen** Checks mit
slice-031-Begründung (verliert die anderen), (iii) per-Verzeichnis-`.clang-tidy`.
§2.4 (kein NOLINT) bleibt in allen Fällen gewahrt. **Near-zero-probability** (alle
13 sind const-**kompilierbar** verifiziert). → **Klausel umformuliert** (Plan §2/§6,
Beleg).

### MED-2 (erster) — Dry-Run-Kommandoform kann still falsche Fundmenge liefern
Empirisch: `-p build -checks=…` gegen die **gebackene `bcad:build`-Quelle** (kein
Bind-Mount) → **const 13 / nodiscard 10** (korrekt, gate-real). Dieselbe Form mit
**Bind-Mount `$PWD` auf einem Host ohne `build/`** → const **37**, disjunkte
Fundstellen (clang-tidy fällt ohne `compile_commands.json` auf geratene Flags
zurück) — **Phantom-Menge**, kein Gate-Realität. Der genutzte Beleg-Lauf
verwendet die **gebackene Form** (13/10). → **Beleg mit Warnhinweis + Sollwerten
13/10 gepinnt.**

### LOW-1 (erster) — §6 charakterisiert const-Fundstellen unvollständig
Gemessen: OCC-Builder ×5, `std::ifstream` **×2** (nicht „einem"), Kern
`structure_edit_service` ×3 (`model::Opening&` → `const&`), `main.cpp`-Composition
×3. 6 von 13 liegen **nicht** bei OCC (3 davon im Hexagon-Kern). Risikoprofil
unverändert (alle verhaltensneutral). → **§6-Wortlaut korrigiert; Beleg führt die
Voll-Liste.**

### LOW-2 (frisch) — `mutableOpening(id)` an `const&` = leichter Code-Smell
Der Minimal-Fix `const model::Opening& target = mutableOpening(id)` ruft einen
„mutable"-Accessor nur zum Lesen. Ein const-`opening(id)`-Accessor wäre
ausdrucksstärker, aber das wäre ein Verhaltens-/API-Umbau (bewusst gemieden).
Nicht blockierend.

### INFO
- **nodiscard-Basis additiv:** der Check flaggt nur die **3 Overrides** in
  `plugins/*`, **nicht** die pure-virtual Basis `plugin.h:20`. Basis-Markierung
  silenced die Overrides **nicht** → **beide** markieren (Plan korrekt). Fixtures
  `abi_mismatch_plugin`/`no_symbols_plugin` überschreiben `name()` nicht → 3 reicht.
- **const-Kaskade:** const auf A kann B const-fähig machen (Folgelauf-Befund);
  `make lint`=0 am Ende fängt jede Kaskade → selbstkorrigierend; finale Edit-Zahl
  kann von 13 abweichen.
- **const compiles-but-breaks real niedrig:** `const std::ifstream` (nur `!in` +
  `.rdbuf()` const) und OCC-Builder (`.Face()`/`.Shape()` const, Build im Ctor)
  kompilieren + verhalten sich identisch; Test-Suite ist ausreichendes Netz.
