## Findings + Disposition

### LOW-1 — §6-Zeilenzahl: „eine oder zwei" weicht von ADR/Index („zwei") ab
ADR-0016 §Konsequenzen + ADR-Index sagen wörtlich „**§6 zwei neue Vertragszeilen
PDF + PNG**"; MR-001 stellt die ADR über die Spec-Schreib-Entscheidung → das
„oder zwei"-Offenlassen weicht von der höher-rangigen Quelle ab.
**Disposition: eingearbeitet.** DoD + Risiko: Default **zwei** Zeilen; kombinierter
Row nur mit expliziter Closure-Begründung.

### LOW-2 — §7-Chirurgie mehrdeutig; Parität zur decided-Nennung fehlt
§7 ist ein Sammel-Bullet (IFC/STEP-STL/DXF **entschieden** + „PDF/PNG bleiben
offen"); zu entfernen ist **nur** der offen-Schwanz, nicht der Bullet. Bei DXF (021a)
gewann §7 eine „DXF entschieden (§1 …)"-Nennung → Parität: „PDF/PNG entschieden
(§1 LH-FA-IO-007.a)" ergänzen statt reiner Streichung.
**Disposition: eingearbeitet.** DoD + Risiko präzisiert (nur Schwanz streichen,
Bullet erhalten, symmetrische decided-Nennung ergänzen).
