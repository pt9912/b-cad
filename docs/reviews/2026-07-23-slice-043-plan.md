## HIGH-1 (blockiert) — `adapter_sink` ist skalar; die geplanten `view/`-Sink-Dateinamen lassen `make a-check` scheitern

**Empirisch belegt** (realer a-check `@sha256:203df7ab…`, `--network none`, read-only, auf einer `src/`-Kopie).
`.a-check.yml` `adapter_sink: adapters/ui/view/mesh_source` ist ein **skalarer Präfix**, der **genau eine** Datei
von Regel B (kein Adapter→Adapter) ausnimmt. Der generische Edge `ui_command → ui_view` reicht **nicht** — mit
den geplanten Namen (`view/plan_source.h`, `view/guide_line_sink.h`, je aus `command/` implementiert):
```
lateral-adapter: 2
  command/edit_drawing_guide_line_sink.h → view/guide_line_sink.h
  command/plan_view_plan_source.h        → view/plan_source.h
exit=1
```
Das `MeshSource`-Muster ist nur grün, weil `mesh_source` registriert ist. **Failure:** der **erste** `make a-check`
scheitert; die DoD-Fitness ist so unerreichbar. Die Plan-DoD-Behauptung „keine neue Allow-Kante nötig" war für die
`edges:`-Liste richtig (`ui_command → ui_view` existiert), übersah aber den **Sink-Carveout**-Mechanismus.
**Auflösung (im Plan §0 eingearbeitet):** **A** `std::function`-Verdrahtung (kein `command/ → view/`-Include →
null Config-Änderung, ehrlich — empfohlen) · **B** `adapter_sink` auf `adapters/ui/view` weiten (empirisch 0
Befunde, aber §2.6-Lockerung → kollidiert mit [ADR-0019](../plan/adr/0019-drw-2d-canvas.md) §Konsequenzen „lockert
keine Allow-Liste", bräuchte Owner/ADR-Re-Eval) · **C** Dateien `mesh_source_*` nennen (**verworfen** — Gate-Gaming).
**Projektinhaber-Entscheidung VOR Start.**

## MED-1 (eingearbeitet) — ACC-002-Beleg via `grabFramebuffer` + Umschalt-Layout; Fix fehlte, Schwere überzeichnet

`main.cpp` `--acc-002-beleg`: `window.show()` → `viewer->grabFramebuffer()`. Ein `QOpenGLWidget`, das im
`QStackedWidget` **nicht** die aktuelle Seite ist, initialisiert beim `show()` seinen GL-Kontext nicht →
`grabFramebuffer` läge Null, Beleg schlägt fehl. **Fix (eingearbeitet):** `stack->setCurrentWidget(viewer)` **vor**
`show()`. **Entlastung (eingearbeitet):** acc-002-beleg ist **manuell, kein Gate**; das **gated** Viewer-Smoke
(`test_viewer_widget.cpp`) baut `ViewerWidget` **standalone** → vom Umbau unberührt → `make gates` bricht über
diesen Pfad nicht.

## LOW-1 (eingearbeitet) — Typ-Mismatch aktives Geschoss

`model::StoreyId` = `enum class : int`, aber `model::StoreyPlan.storey_id` = plain `int` → Filter braucht
`static_cast<int>(activeStoreyId) == sp.storey_id` (sonst leere Zeichnung). Im Plan (DoD-1) benannt.

## LOW-2 (eingearbeitet) — aktive Ebene: Id vom Root injizieren, nicht raten

`addGuideLine` lehnt bei unbekannter Ebene ab. Der Root muss die Hilfslinien-Ebene via `addLayer` anlegen und die
**zurückgegebene** `LayerId` injizieren (keine hartkodierte `LayerId{1}`), Reihenfolge Ebene→Id→Sink→subscribe VOR
Interaktion. Im Plan (DoD-4) benannt.

## Verifiziert (INFO — Plan-Annahmen halten am Code)

- **INFO-1** `EditDrawingPort::addGuideLine(const GuideLine&)` vergibt die `GuideLineId` **selbst**
  (`structure_edit_service.cpp` `next_guide_line_id_`); Ablehnung bei Entartung/unbekannter Ebene/unbekanntem
  Geschoss — exakt die geplante Boundary-AK.
- **INFO-2** `service.building()` ist public Getter, `building_.guide_lines` public → der display-freie AK-Anker ist real.
- **INFO-3** `test_viewer_widget.cpp` belegt manuelle `QApplication` + `QMouseEvent` + `QApplication::sendEvent`;
  für die zustands-verankerte AK genügt `resize()` (Viewport für `screenToModel`) ohne `show()`/GL — `paintEvent`
  muss nicht feuern.
- **INFO-4** `.a-check.yml` erlaubt `ui_view → {model, ports_driven}`; `ui_view → ports_driving` **nicht**. Der
  Canvas als `ModelChangedPort`-Beobachter (`view/`) ist erlaubt (Stub-Canvas mit `driven/model_changed_port.h` →
  0 Befunde). Der einzige Bruch ist die Sink-**Datei-Registrierung** (HIGH-1), nicht die Kanten-Richtung.
- **INFO-5** `plan_view.h` liefert Segmente je Geschoss in mm + BBox + `has_geometry`; leeres Modell → degenerierte
  Box `(0,0,0,0)` → Zoom-Fit muss den Fall abfangen (Plan-DoD benennt es).
- **INFO-6** Selbst-Refresh + Beobachter-Refresh sind beide queued `update()` (Qt-coalesced); `addGuideLine` ist
  `op`-los → `onModelChanged` feuert für Hilfslinien nicht → keine Reentranz/Doppel-Repaint.
