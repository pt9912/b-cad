## M1 (MED) — AK-Fehlzuordnung: „DRW-006-Negative" ist die falsche AK für unsichtbar→fehlt

**Plan-Behauptung:** „DRW-005-Happy … **und DRW-006-Negative** (unsichtbare Ebene → keine Hilfslinie
im Artefakt)"; wiederholt im §6-Risiko + der DoD-Suite-Benennung.
**Widersprechende Quelle:** `spec/lastenheft.md` — **DRW-006-Negative** ist der **Löschschutz**-Fall
(„Ebene, die noch von einer Hilfslinie referenziert wird … Löschen abgelehnt"), **bereits in
[slice-032b](../plan/planning/done/slice-032b-drw-impl.md) geschlossen** (ADR-Index). Das
invisible→absent-Verhalten ist **DRW-005-Negative** („Hilfslinie auf einer Ebene, die auf unsichtbar
geschaltet ist … erscheint keine Hilfslinie dieser Ebene im Artefakt") **+** die
**DRW-006-Happy-Sichtbarkeits-Klausel** („Die Sichtbarkeit bestimmt, ob die Hilfslinien dieser Ebene
im 2D-Grundriss-Export erscheinen").
**Impact:** systematisch (≥2×), propagiert in DoD-Test-Benennung + Closure/CHANGELOG-Traceability — der
Slice würde eine bereits geschlossene AK zu schließen behaupten und die real geschlossene falsch
etikettieren. Das **Verhalten** ist korrekt beschrieben; nur die Zitierung ist falsch → nicht blockierend.
**Fix (eingearbeitet):** überall auf **DRW-005-Negative + DRW-006-Happy-Sichtbarkeits-Klausel**
umetikettiert; die „Fehlt"-Tests entsprechend benannt (nicht DRW-006-Negative).
