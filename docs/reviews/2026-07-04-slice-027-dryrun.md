# slice-027 — clang-tidy-Familien Dry-Run-Bestandslauf (Beleg-Matrix)

**Datum:** 2026-07-04 (Plan datiert 2026-07-03; Ausführung slipte auf 07-04).
**Zweck:** Evidenz-Beleg für die kuratierte `.clang-tidy`-Familien-Erweiterung
(slice-027). Ablage unter `docs/reviews/` ist die benannte Ausnahme (Evidenz
statt Review, Plan-Review-LOW-4; `ids`-exempt-Klasse).

## Reproduktion (exaktes Kommando, Review-LOW-2)

Gepinnte Toolchain ([ADR-0004](../plan/adr/0004-toolchain-dependency-pinning.md),
clang-tidy **21.1.8**), Lauf im Container gegen das gebaute `bcad:build`-Image
(`compile_commands.json`), Scope `src/` + `plugins/` (43 `.cpp`),
`HeaderFilterRegex: '^src/'` aus `.clang-tidy` geerbt (gate-repräsentativ,
Plan-Review-LOW-1). **Ein** kombinierter Lauf aller 7 Familien (TUs einmal
kompiliert), danach Bucketing nach Check (statt 7 Einzelläufe — selbe Matrix,
effizienter):

```
docker run --rm bcad:build clang-tidy -p build \
  --extra-arg=--gcc-install-dir=/usr/lib/gcc/x86_64-linux-gnu/15 \
  -checks='-*,cert-*,readability-*,performance-*,modernize-*,misc-*,cppcoreguidelines-*,portability-*' \
  $(find src plugins -name '*.cpp')
```

Ergebnis: **1630 Befunde, 0 Parse-/Compile-Fehler.** (`docker run <image> <tool>`
ist regelkonform — clang-tidy im gepinnten Image, kein Host-Tooling;
PreToolUse-Guard erlaubt es, [AGENTS.md §2.9](../../AGENTS.md).)

## Kurations-Bilanz

- **Aktiviert (Befunde gefixt): 8 Checks, 10 Befunde** (wertvoll, wenig,
  verhaltensneutral). Plus: durch Aktivieren der 7 Familien mit Exclude-Liste
  sind **alle 0-Befund-Checks** der Familien scharf (Falsifizierbarkeits-
  Invariante, Plan-Review-INFO-2).
- **Ausgelassen: 24 Check-Namen** mit Befunden — je begründet (Alias/Deckel/
  Idiom/Stil/false-positive), im `.clang-tidy`-Kommentar mit Slice-Bezug.
- **Kumulativer Fix-Budget-Escape (~60):** klar unterschritten (10 Fixes) —
  lean, low-risk; die zurückgestellten wertvollen Kandidaten (const-correctness
  14, nodiscard 10, inconsistent-param-name 19 = 43) sind **benannte
  Folge-Kandidaten** einer späteren lint-Härtung.
- **HeaderFilterRegex-Vorbehalt:** `plugins/`-Header liegen außerhalb `^src/` —
  wie beim Gate; Matrix ist gate-repräsentativ. `plugins/`-`.cpp` sind gescannt.
- **Idiom-Präzisierung ggü. Plan §6:** der `reinterpret-cast`-Kollisionspunkt
  ist **SQLite-C-API** (nicht dlsym wie vermutet); `owning-memory` trifft wie
  erwartet die Plugin-ABI. Beide bleiben ausgelassen.
