## Verifiziert (kein Defekt)

- **A — Determinismus-Tabelle korrekt.** IFC (`ifc_spf_writer.cpp:18,34` Sentinel-TS), DXF (`dxf_export_adapter.cpp:43-49`
  nur `$ACADVER`, keine Handles/Datum), PDF (`pdf_writer.cpp:90-161` kein `/CreationDate`/`/ID`/`/Producer`), PNG
  (`png_writer.cpp:169-171` kein `tIME`) → **byte-stabil**. STEP korrekt als **nicht-deterministisch** (`step_export_adapter.cpp:115`
  nackter `STEPControl_Writer`, OCC brennt Wall-Clock in `FILE_NAME`); der geplante Header-Fix (`APIHeaderSection_MakeHeader(writer.Model())`)
  ist in OCC 7.9.2 vorhanden + `writer.Model()` vor `Write` (`:125`) zugreifbar → **machbar**. STL Header fest, Geometrie OCC-versions-resident.
- **B — Laufzeit-Mechanik hält (die „geht das überhaupt"-Frage).** `.devcontainer/Dockerfile` `build`-Stage
  `WORKDIR /src` + `COPY . .`; `test`/`coverage-check` sind `FROM build` → `BCAD_TEST_GOLDEN_DIR="/src/tests/adapters/golden"`
  ist zur Testlaufzeit lesbar (COPY, kein Bind-Mount). Die Golden-Tests laufen damit **in `make gates`** (nur
  `golden-check`-Regen ist CI-only). Korrekt.
- **C — komplementär, nicht redundant.** Die vorhandenen Orakel sind Struktur-**Decoder** (PDF xref/`/Length`/
  Objektgraph; PNG Inflate/Adler/CRC), **kein** Byte-Vergleich → Byte-Golden fängt Encoder-Drift bei unveränderter
  Struktur. Echte neue Fehlerklasse.
- **D — DXF-Import-Realismus korrekt.** `dxf_import_adapter.cpp:108-126` liest `LINE`/`LWPOLYLINE`, überspringt
  3D/anderes graziös → 2D-`LINE`-Fixture importiert sauber, 3D → 0 Wände (kein Wurf).
- **E — Lizenzen plausibel.** buildingSMART Sample-Test-Files CC-BY-4.0; ezdxf/dxf-parser/ixmilia MIT.

## MED-1 (eingearbeitet) — IFC-Import ist enger als „Extrusions-Wände" und WIRFT (nicht Skip)

`ifc_import_adapter.cpp:342-345` (kein 'Axis' → `throw`) + `:237-239` (keine Spatial-Containment → `throw`) → in
`read()` (`:388-393`) **`E-IO-003` Ganzdatei-Ablehnung**. Die Pflicht-Schlüssel je Wand sind **'Axis'-`IFCPOLYLINE`
+ `IFCRELCONTAINEDINSPATIALSTRUCTURE`**; `IfcExtrudedAreaSolid` liefert nur die Höhe (Default-Fallback `:303-304`).
**Failure:** eine reale `Body`/`SweptSolid`-only-Sample-Datei (häufig) → `E-IO-003`, der geplante Test (`walls>0;
kein Wurf; Skip`) trifft das falsche Muster. **Fix (eingearbeitet):** 044b-Fixture so **kuratieren**, dass **jede**
`IfcWall` Axis + Containment trägt (ggf. minimale konforme IFC gezielt erzeugen); Test-Erwartung: b-cad **nimmt an**
(kein `E-IO-003`). **Blockiert 044b-Start, nicht 044a.**

## MED-2 (eingearbeitet) — `golden-regen` „über das Binary" driftet vom festen Test-Modell

Das Binary baut `buildAcc001KernDemo` (`main.cpp:111`), nicht `goldenModel()` → Regen über das Binary driftet
garantiert. Zudem taugt `schema-regen`s stdout-Redirect (`Makefile:148-153`) nicht für 6 Dateien inkl. binärer
STL/PNG. **Fix (eingearbeitet):** dedizierter Generator, der die **`goldenModel()`-TU mit dem Test teilt** (LOW-3),
schreibt über einen **writable Mount** (Muster `acc-002-beleg`); „über das Binary" gestrichen.

## LOW (eingearbeitet)
- **LOW-1:** `BCAD_TEST_PLUGIN_DIR` zeigt aufs **Build**-Dir (`tests/CMakeLists.txt:58-59`), Golden brauchen das
  **Quell**-Dir (`${CMAKE_CURRENT_SOURCE_DIR}`) — anderes Muster, nicht „analog" verkürzen. Im Plan präzisiert.
- **LOW-2:** STEP-Body-Byte-Stabilität (OCC-Entity-Ordering) run-zu-run unbelegt → vor STEP-Golden-Commit
  `golden-regen` **zweimal** + `diff`. In DoD.
- **LOW-3:** `goldenModel()` als **eine** geteilte TU (Test + Regen-Tool linken sie) — explizites Deliverable
  (heute je Export-Test ein eigenes `sampleBuilding()`). In DoD.
