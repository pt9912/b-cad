## MED

- **M1 — Gate-Member-Kriterium in MR-018 von „hermetisch/git-frei" (MR-017-Basis) auf „range-frei"
  verfeinern; sonst latenter Kriteriums-Drift zwischen zwei immutablen MR-Einträgen.** MR-017
  begründet `planning` als Gate-Member wörtlich mit **Hermetik ohne git** („nur Arbeitsbaum, kein
  git"). `tracked` erfüllt das **nicht** — es liest den git-Index, **fail-closed ohne `.git`** (belegt:
  nach `git add` eines Ziels kippt `target-untracked` → 0, ohne Commit). Erbt der immutable MR-018-Text
  die „**wie planning**"-Verkürzung, entsteht ein stiller Widerspruch (MR-017: „Gate-Member weil
  git-frei"; MR-018: „wie planning" bei einem git-lesenden Modul). **Fix:** MR-018 formuliert das
  Kriterium explizit — **Gate-Membership = „keine Range" (deterministischer Einzel-Snapshot), NICHT
  „git-frei"**; `tracked` = **erster Gate-Member, der `.git` liest** (range-frei), fail-closed ohne
  `.git` (unkritisch, da `docs-check` `.git` mountet). Plus die **CI-Vakuität** als eigentliche
  Rechtfertigung: auf `index == HEAD` (frischer CI-Checkout) ist alles getrackt → `tracked` in CI
  **immer 0**; Wert nur lokal auf dem schmutzigen Arbeitsbaum → **CI-only wäre nutzlos-grün, Gate-Member
  die einzig werthaltige Platzierung**.

## LOW / INFO

- **L1 — Index-Semantik in MR-018 benennen (gestagt = getrackt).** Ein nur **gestagtes** (nicht
  committetes) Ziel zählt als getrackt (Index = Proxy für „was der nächste Commit trägt"). Das
  theoretische Loch (staged→grün, dann selektiv nur das Doc committen) fängt `links` auf dem
  resultierenden Commit (`target-missing`). Kein Bypass entkommt. **Behoben (benannt).**
- **L2 — `make gates` gewinnt seinen ersten harten `.git`-abhängigen Member.** Alle bisherigen Member
  (inkl. `planning`) laufen auch ohne `.git`; `tracked` fail-closed ohne `.git`. Im Devcontainer-/CI-
  Mount unkritisch (`.git` da), aber ein `.git`-loser Kontext (Source-Tarball) fail-closte `make gates`.
  Ein-Zeilen-Notiz (Muster MR-013-Bind-Mount-Ausnahme). **Behoben (benannt).**
- **L3 — „frischer Klon" kollidiert terminologisch mit [MR-005](../../harness/conventions.md).**
  `conventions.md:228` nutzt „frischer Klon" für die **Working-Tree-Hash-Restlücke** (sha256 über
  Datei-*Inhalte*). MR-018 muss die Abgrenzung **im Text tragen**: MR-005 = Nachweis-Hash über Inhalte;
  `tracked` = Index-Mitgliedschaft der Referenz-**Ziele**. (H1-Sweep: **keine** vorbestehende Regel →
  „neue Regel, eine Heimat" korrekt.) **Behoben.**
- **L4 — Nebenclaim „Symlink-Ziele prüft tracked nicht" ist unverifiziert (d-check-Interna).** Der
  Verzeichnis-Teil ist indirekt belegt (Korpus hat Verzeichnis-Links, Baseline 0). b-cad hat keine
  Symlink-Ziele → folgenlos. Als „aus d-check-Doku, nicht b-cad-verifiziert" kennzeichnen. **Behoben.**

## Verifiziert / solide (reproduziert; Baum je zurückgesetzt)

- **Baseline:** `make doc-tracked` = 0 (187); `make docs-check` (Default) = 0. Ist-Zustand konsistent.
- **Positiv (isoliert, ein Lauf):** Wegwerf-Doc → untracktes existierendes Ziel ⇒ **`target-untracked`**,

- **Kein Doppelbefund (links + tracked aktiv):** exakt **2** distinkte Befunde — `target-missing` (links)

  ist alles getrackt ⇒ `tracked` = 0. `tracked` ist ein **lokaler Pre-Handoff-Wächter**, in CI
  vakuum-grün → Gate-Member die einzig werthaltige Platzierung; löst zugleich jede Commit-Reihenfolge-

  generierten Ordner (`build`/`out`/`coverage`/`cmake-build` grep leer), kein `build/`-Dir.
- **Keine Regression:** Default-Set **+** `tracked` auf cleanem Repo = 0 ⇒ Aufnahme additiv.
- **Steering/Form:** MR-018 korrekte Nummer; Frontmatter anforderungsfrei; „zweiter Fahrplan-Gate-Member
