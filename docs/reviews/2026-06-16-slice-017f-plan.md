## Negativbefunde (geprüft, ohne Befund)

- **Lastenheft EVL-004/005/006-Wortlaut** (`lastenheft.md:669-691`): Plan-Claims treu — EVL-004 „Menge (Fläche **bzw.** Volumen)" (die Volumen-Wahl liegt im AK-Rahmen, **kein** Lastenheft-Edit nötig), Boundary „ohne Material → nicht gruppiert, kein Fehler"; EVL-005 „Türen, Anzahl + Breite/Höhe"; EVL-006 „Fenster, Anzahl + Breite/Höhe/Brüstung".
- **`effectiveMaterial`-Reuse** (`evaluate_port.h`, `structure_edit_service.cpp`): `effectiveMaterial(WallId)`/`(SlabId)`/`(RoofId)` → `std::optional<model::Material>` (Override, total). Direkt nutzbar.
- **`volume_geometry`-Reuse** (`volume_geometry.h`): `wallNetVolumeMm3(Wall, walls, openings)` + `slabNetVolumeMm3(Slab)` (mm³, total); `volume()` teilt bereits durch `kMm3PerM3=1e9`. Reuse-Muster exakt.
- **Opening-Modell** (`opening.h`, `building.h`): `kind` (Door/Window) + `width_mm`/`height_mm`/`sill_height_mm` (0 für Türen); `Building.openings` mischt Türen+Fenster. DoorLine/WindowLine baubar; `kind`-Filter korrekt.
- **`MaterialLine` trägt `model::Material`** (`material.h`): pure Werte, kopierbar, framework-frei — trägt Kennwerte für die spätere Kosten-Auswertung.
- **`ADR-0012`/0001-Konsistenz:** drei Listen sind read-only-Aggregationen (kein `op`, keine Mutation); `wallNetVolumeMm3` ist Kern-analytisch (kein `GeometryKernelPort`, kein `Solid.volume_mm3`). Reine Kern-Query, `const`-Methoden.
- **`MR-009` n/a:** keine neue Geometrie (Reuse der 017c-code-reviewten Helfer); keine neue Korrektheits-Fläche. Sound.
- **Determinismus:** `std::map<MaterialId,…>` — `MaterialId` ist `enum class : int` (`operator<`), Muster `std::map<WallId,…>`/`<StoreyId,…>` im Bestand. Material-Gruppen nach Id, Öffnungen in Modell-Reihenfolge = deterministisch.
- **Lastenheft unberührt / `MR-010`:** EVL-AK seit 017a auf Niveau; Header 0.1.7 unberührt; Spec-Edit nur spezifikation.md §1 + Historie. Korrekt.
- **Sizing (modul-05):** 4 Werttypen + 3 Port-Methoden + Service + Test — am oberen Rand, aber **hohe Reuse-Dichte** (Tür/Fenster trivial; Material reuse `effectiveMaterial`+`volume_geometry`), eine Sub-Area, eine Review-Sitzung. Akzeptabel.
